<objective>
Strip all per-tool history capture from the Hash, JWT, Base64, and URL tools. Each tool's ViewModel currently takes an `onSaveHistory: (HistoryEntry) -> Void` closure, stores it, and calls it after a transform; each View wires that closure from an `@Environment(HistoryStore.self)`. Remove the closure param, the stored property, every call site, the environment read, and the preview `.environment(HistoryStore())` — for all four tools.

This is the security-sensitive group: Hash and JWT carry explicit INFRA-09 "HMAC key / JWT secret never reaches onSaveHistory" comments. Removing history makes those hazards moot; keep the key/secret View-local as-is (do not change verification behavior).

Purpose: These four tools stop referencing the history subsystem so it can be deleted in Wave 3.
Output: Four tools that build and function with zero history references.
</objective>

<pattern_note>
Two wiring shapes exist across the 13 tools. This plan covers:
- Hash: Wrapper-pattern — HashDefinition.swift has a private `HashViewWrapper` struct that reads `@Environment(HistoryStore.self)` and calls `HashView { entry in historyStore.save(entry) }`. HashView has `init(onSaveHistory:)`.
- JWT, Base64, URL: Environment-in-View — the View itself reads `@Environment(HistoryStore.self)`, builds the ViewModel with `onSaveHistory: { [historyStore] entry in historyStore.save(entry) }`, and the Definition just calls `AnyView(SomeView())` with no closure.

Both shapes end at the same place: remove the ViewModel's param + call sites, then remove whatever wiring fed it.
</pattern_note>
</context>
