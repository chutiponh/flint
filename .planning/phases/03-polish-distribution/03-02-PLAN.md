<objective>
Deliver DIST-02: every tool accepts drag-and-drop of text files; binary tools (Base64, Hash) accept any file. A dropped file on an open tool loads into THAT tool (D-04). A drop on the launcher reads the file text, runs `detect()`, and routes to the best tool (mirrors Services D-02, reusing `openLauncherWithStagedText` from plan 03-01 for the no-match staging). Text-only tools reject binary/oversized files with the existing `WarningBannerView` (D-06); binary tools stream any file off-main via the existing chunked pipeline. A whole-surface drag-over overlay (D-05) provides feedback with no permanent drop zone.

Purpose: Make file content a first-class input path alongside paste and Services, without regressing the shipped large-file-hash capability (no blanket size cap — D-06).
Output: `DropOverlayView`, a shared `FileDropHandler`, `.onDrop` + overlay on all 12 tool views and the launcher, and a `loadFile(url:)` entry point added to Base64ViewModel.
</objective>

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Filesystem → Flint via drop | Arbitrary file (binary, oversized, non-UTF-8, alias URL) crosses into tool input or the file pipeline |
| NSItemProvider internal queue → @MainActor | Drop completion runs off-main; state mutation must hop to MainActor |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-04 | Tampering | Binary file dropped into text tool | mitigate | UTF-8 decode attempt; throw → WarningBannerView rejection; file content is read, never executed. |
| T-03-05 | Denial of Service | Oversized file dropped into text tool | mitigate | 5MB text-tool size guard via fileSizeKey → reject with "too large" copy; binary tools intentionally uncapped (use existing streaming pipeline, off-main). |
| T-03-06 | Denial of Service | UI freeze on large binary drop | mitigate | Binary drops call existing off-main chunked pipeline (startFileHash/encodeFileChunked); never read on @MainActor. |
| T-03-07 | Information Disclosure | file-reference/alias URL leaks real path | accept | url.lastPathComponent used for display only; Foundation resolves alias URLs for read; low risk, non-sandboxed local app. |
| T-03-SC | Tampering | npm/pip/cargo installs | mitigate | No package installs in this plan. N/A. |
</threat_model>
