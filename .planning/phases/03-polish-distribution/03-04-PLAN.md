<objective>
Deliver the onboarding half of DIST-03: a single focused welcome window shown once on first run (D-07). Its real job is the "where did it go?" problem of a no-Dock-icon menubar app — it points at the menubar wrench icon, teaches the `⌘⇧Space` global hotkey, and offers one CTA to enable Launch at Login (reusing the existing `PreferencesStore.launchAtLogin` SMAppService path — no new launch code). Dismissing sets a `hasSeenOnboarding` flag so it never reappears. The window uses the existing `WindowCoordinator` activation-policy dance to appear above the frontmost app, and the first-run gate is a cheap UserDefaults bool so cold start is not regressed (RESEARCH Pitfall #6/#7).

Purpose: A fresh user must know Flint is alive in the menubar and how to summon it; this is the single highest-value onboarding signal.
Output: `hasSeenOnboarding` pref key, `OnboardingWindowView`, `WindowCoordinator.openOnboarding()`, an onboarding `WindowGroup` in FlintApp, and the first-run gate.
</objective>
