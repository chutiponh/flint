<objective>
Deliver DIST-01: a single macOS Services menu entry ("Open in Flint") that takes text selected in any app, runs `ToolRegistry.detect(from:)`, and opens the best-matched tool pre-filled via `ToolSeed` — or, on no match, opens the launcher with the text staged in the search field.

This plan also migrates the project from `GENERATE_INFOPLIST_FILE=YES` to a manual `Info.plist`, because `NSServices` is an array-of-dictionaries that cannot be expressed as a scalar `INFOPLIST_KEY_*` build setting (RESEARCH Pitfall #1). This manual Info.plist is the foundation that the Sparkle plan (03-04) extends with `SUPublicEDKey`/`SUFeedURL`.

Purpose: Services is the first "send to Flint from anywhere" surface and the substrate-reuse proof — it routes through the FROZEN `detect()` + `ToolSeed` mechanism with one intentional divergence (D-02: skip the detection banner, auto-open).
Output: Manual Info.plist with NSServices, AppDelegate service registration, FlintServiceProvider handler, two new WindowCoordinator routing methods, and the FlintApp notification receiver that performs the seed-and-open dance.
</objective>

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Any app → Flint via Services pasteboard | Untrusted text of arbitrary size/content crosses into Flint's detection + tool input |
| AppKit service thread → SwiftUI @MainActor | Service handler runs off-main; all UI/state mutation must hop to MainActor |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-01 | Tampering | FlintServiceProvider pasteboard text | mitigate | Text is read, never executed; routed only into detect() and tool input fields. Tools already validate gracefully (INFRA-17). |
| T-03-02 | Denial of Service | Oversized Services text (1MB+ selection) | mitigate | Cap incoming text at 1MB in openInFlint; return silently if exceeded (mirrors clipboard oversized guard). |
| T-03-03 | Spoofing/Injection | NSMessage selector mismatch | mitigate | NSMessage in Info.plist asserted equal to @objc selector base name `openInFlint` via PlistBuddy + grep gate (Task 1/2). |
| T-03-SC | Tampering | npm/pip/cargo installs | mitigate | No package installs in this plan (Sparkle is plan 03-04). N/A here; legitimacy gate deferred to 03-04. |
</threat_model>
