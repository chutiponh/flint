Purpose: Prove the whole distribution + update loop locally so the first public release is correct and updatable.

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Build machine → distributed artifact | Signing/notarization integrity determines whether users can trust and run the app |
| Network appcast/DMG → installed app | Update artifacts must be EdDSA-verified and Gatekeeper-valid |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-14 | Tampering | Sparkle XPC services corrupted by codesign --deep | mitigate | Release script uses Xcode Archive → Export (Developer ID), which re-signs XPC in correct order; script asserts no `codesign --deep`; checkpoint runs `codesign --verify --deep --strict`. |
| T-03-15 | Spoofing | Unsigned/un-notarized app blocked or distrusted | mitigate | notarytool submit --wait + stapler staple; checkpoint verifies via spctl + stapler validate (Gatekeeper accept). |
| T-03-16 | Privilege Escalation | get-task-allow present in release build | mitigate | Release config uses Flint-release.entitlements (no get-task-allow, Hardened Runtime on); Archive export strips it. |
| T-03-17 | Information Disclosure | EdDSA private key leaked during appcast signing | mitigate | generate_appcast reads key from login Keychain only; never echoed/written by scripts; backup off-machine per DISTRIBUTION.md. |
| T-03-SC | Tampering | create-dmg install (npm/brew) | mitigate | create-dmg 8.1.0 locked in CLAUDE.md (authoritative); installed via brew/npm in Task 1 (human checkpoint); no [SLOP]/[SUS] packages introduced. |
