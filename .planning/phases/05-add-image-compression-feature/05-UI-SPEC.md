# Phase 5 — UI Design Contract

> Visual and interaction contract for Phase 5: Image Compressor tool.
> Generated by gsd-ui-researcher. Consumed by gsd-planner and gsd-executor.
>
> **Platform note:** Native macOS SwiftUI application (macOS 14+). No web stack, no
> shadcn, no Tailwind. All design tokens are SwiftUI/AppKit system APIs in logical
> points (1pt = 1px at @1x; Retina renders @2x).
>
> **Continuity note:** The design system established in Phases 1–4 (01/02/03/04-UI-SPEC.md)
> is the authoritative baseline. This spec adds only what Phase 5 introduces: a new tool
> (`ImageCompressView`) living inside the existing 480×600pt popover, plus its net-new
> surfaces — the results table (thumbnail + original→new size + % saved + per-row
> progress, D-09), the quality slider + presets (D-04), the lossless-format gating
> (D-05), and the new copy. The tool MUST visually match the existing tools and follow
> the Hash tool's layout idiom (`Tools/Hash/HashView.swift`). Nothing here overrides the
> prior contracts.

## Spacing Scale

Carried forward from Phase 1/4 (01-UI-SPEC.md, 04-UI-SPEC.md). All values in logical points, multiples of 4. Match the Hash tool layout idiom exactly.

| Token | Value | SwiftUI Usage in this phase |
|-------|-------|------------------------------|
| xs    | 4pt   | Thumbnail-to-text gap inside a row, size-arrow inline spacing |
| sm    | 8pt   | Section heading-to-content gap, preset button row spacing, results table row internal spacing, VStack section spacing |
| md    | 16pt  | Default tool content `.padding()` (matches `HashView` `VStack(spacing: 16)` + `.padding()`); spacing between major sections (drop area / controls / results) |
| lg    | 24pt  | Reserved — not used in 480×600 popover this phase |
| xl    | 32pt  | Reserved — not used this phase |
| 2xl   | 48pt  | Reserved — not used this phase |
| 3xl   | 64pt  | Reserved — not used this phase |

**Phase 5 exceptions / additions (Claude's discretion per CONTEXT.md — spec'd prescriptively):**

- **Popover fixed size: 480×600pt** (INFRA-01 — locked, unchanged). The entire tool must fit and scroll within this width. The results table is constrained to this width — design columns for a ~448pt usable content width (480 − 16pt padding each side).
- **Thumbnail size: 40×40pt** (logical), `cornerRadius(4)`, `.aspectRatio(contentMode: .fill)` clipped to a `RoundedRectangle(cornerRadius: 4)`. 40pt chosen to stay row-height-compact in the constrained popover while remaining recognizable; smaller than a full preview (no before/after comparison — D-10).
- **Results table row height: 56pt** minimum (40pt thumbnail + 8pt vertical padding top/bottom). Rows separated by `Divider()` (`Color(NSColor.separatorColor)`), not custom borders.
- **Quality slider + preset row block:** preset `Button`s in an `HStack(spacing: 8)`; slider with 8pt vertical padding above/below within its labeled section.
- **Results section outer container:** `.padding(10)` + `.background(.quaternary.opacity(0.3))` + `.cornerRadius(8)` — identical to `HashView`'s `fileHashSection` container idiom (HashView.swift L250–252), for visual continuity with the file-hash surface.
- **Inline 10pt sub-labels** (e.g. "PNG · lossless" format tag) — off-scale inherited exception, locked from Phase 3/4 (10pt sub-labels permitted).
- **12pt WarningBannerView text** — off-scale inherited exception, locked (WarningBannerView is reused unchanged).

**Two weights only:** Regular (400) and Semibold (600). No bold (700), no medium (500).

**Phase 5 typography notes:**

- Tool title "Image Compressor": Subhead (15pt semibold), `Color.primary`. Mirrors `HashView`'s `Text("Hash Generator").font(.headline)` — use 15pt semibold to stay consistent with the Phase-4-declared Subhead role.

- Format tag (e.g. "PNG · lossless"): 10pt regular `.secondary` (off-scale inherited exception).
- Empty-state copy: Body 13pt `.secondary`, centered (matches the Phase-4 D-05 empty-state rule).

## Color

Carried forward from Phases 1–4. All SwiftUI semantic colors — no hardcoded hex (INFRA-14).

| Role | SwiftUI Value | Usage in this phase |
|------|---------------|---------------------|
| Dominant (60%) | `Color(NSColor.windowBackgroundColor)` | Tool surface / popover background |
| Secondary (30%) | `Color(NSColor.controlBackgroundColor)` / `.quaternary.opacity(0.3)` | Results section container, per-row passive background |
| Accent (10%) | `Color.accentColor` | See reserved-for list below |
| Success | `Color.green` | Positive `% saved` value text only (the hero metric) |
| Warning | `Color.yellow.opacity(0.15)` background | `WarningBannerView` `.warning` — non-image / "grew slightly" rows |
| Destructive / Error | `Color.red` | `WarningBannerView` `.error` — hard write failure; "Cancel" button label (matches `HashView` L230 Cancel `.foregroundStyle(.red)`) |
| Text primary | `Color.primary` | Filenames, tool title, slider value, size figures |
| Text secondary | `Color.secondary` | Section labels, format tags, disabled-slider state, neutral/negative `% saved` |
| Separator | `Color(NSColor.separatorColor)` | Row dividers in the results table |

**Accent (`Color.accentColor`) reserved for (explicit list — no other use in this phase):**

1. **`DropOverlayView`** while a drag is over the surface — `Color.accentColor.opacity(0.08)` fill + 2pt accent border + accent `doc.fill.badge.plus` glyph (reused component, unchanged).
2. **Quality slider track tint** — the SwiftUI `Slider` minimum-track uses the system accent automatically; this is the only accent in the active controls. When the slider is **disabled** for lossless formats (D-05), it renders in `.secondary`/system-disabled tint (no accent), reinforcing the "not applicable" state.
3. **Preset button selected state** — the preset whose value currently matches the slider renders `.borderedProminent` (accent fill); non-active presets render `.bordered` (no accent). Selecting a preset sets the slider; manually moving the slider off a preset value clears the prominent state on all presets.

**Accent is NOT used for:** the results table, thumbnails, size figures, `% saved` values (those use green/secondary), section headings, the "Compress" trigger is implicit (drop-driven), or any passive surface.

**Phase 5 new color semantics:**

- **`% saved` value coloring (D-09 hero metric):**
  - Positive savings (new < original): `Color.green`, format `"−42%"` (note the savings are shown as a reduction).
  - Zero or negative (new ≥ original — common for re-encoded PNG/TIFF, Pitfall 2): `Color.secondary`, format `"+3%"` or `"0%"`. Surface neutrally — this is NOT an error, just honest reporting.
- **Lossless-format gating (D-05):** when a row's format is PNG/TIFF, the quality slider does not apply. Render the format tag "PNG · lossless" and, when PNG/TIFF inputs are present, show a `.secondary` helper line under the slider: see Copywriting. The slider control itself is disabled (`.disabled(true)` + `.secondary` appearance) only when the dropped batch is **entirely** lossless; in a mixed batch the slider stays enabled (it applies to the lossy members) and each lossless row shows the "lossless" tag.
- **Per-row progress indicator:** in-progress rows show a `ProgressView()` (indeterminate circular spinner, `.scaleEffect(0.6)`) with the system accent tint, replacing the size/savings figures until the row finishes. No determinate per-row bar (each image is a single atomic encode — there is no sub-image progress to report; determinate progress is at the batch level via the completed-row count).

## Interaction Contracts

These are the behavior rules the executor must implement exactly. Each maps to a CONTEXT.md decision; net-new surfaces with no in-codebase analog are flagged and pull from RESEARCH.md patterns.

### Drop Surface — D-01 (reuse Hash idiom, extend to multi-file)

- `.onDrop(of: [.fileURL], isTargeted: $isDragTargeted)` on the tool's root `ScrollView` (same modifier position as `HashView` L45).
- **Multi-file (net-new vs Hash):** `HashView` reads `providers.first` only. This tool MUST iterate **all** providers and join their async `loadItem` loads via a `DispatchGroup`, then call `viewModel.compress(urls:quality:)` once on `.main`. Use the verified snippet in 05-RESEARCH.md Code Examples (L391–413).
- While dragging: `.overlay { if isDragTargeted { DropOverlayView(label: "Drop images to compress") } }` with `.transition(.opacity.animation(.easeOut(duration: 0.15)))` — identical to `HashView` L56–61, label changed.
- Optional secondary affordance: a "Choose Images…" `Button` using `NSOpenPanel` with `allowsMultipleSelection = true`, `canChooseFiles = true`, `allowedContentTypes = [.png, .jpeg, .heic, .tiff]` (mirrors `HashView.selectAndHashFile` L257–268 with multi-select on). Optional this phase — drag-drop is the primary path.
- **Post-drop rejection (INFRA-17):** a dropped non-image / corrupt file produces a failed row, surfaced as a `WarningBannerView(severity: .warning)` inline in that row — never a crash, never a drag-time rejection style (DropOverlayView has a single valid drag state by design).

### Quality Slider + Presets — D-04 / D-05 (net-new control, no analog)

- **Slider:** SwiftUI `Slider(value: $quality, in: 0...100, step: 1)`. Label "Quality" at Body 13pt to the left; current value shown as "{n}%" at Body 13pt `.primary` to the right (e.g. "80%"). The UI value is 0–100; the Transformer maps `quality / 100.0` to the ImageIO 0.0–1.0 key (05-RESEARCH.md Code Examples).
- **Presets (3 buttons, Claude's discretion — spec'd):**
  | Preset | Slider value | Rationale |
  |--------|--------------|-----------|
  | **Web** | 60 | Smaller files, fine for screen display |
  | **Email** | 75 | Balanced quality/size (default selected on first open) |
  | **Max** | 95 | Near-lossless lossy quality |
  - Default on first open: **Email (75)**.
  - Clicking a preset sets the slider to its value. The slider **stays adjustable** after a preset (D-04) — moving it manually clears the active-preset prominent state.
  - Active preset (value == current slider) renders `.borderedProminent`; others `.bordered`.
- **Persistence (MEMORY.md pitfall):** if the quality value persists across launches, bind the control to `@AppStorage("imageCompressQuality")` with a stable key — do NOT bind to a computed `PreferencesStore` property (writes drop). Default value 75.
- **Lossless gating (D-05):**
  - Detect lossy vs lossless per format (JPEG/HEIC = lossy honors quality; PNG/TIFF = lossless, quality not applicable).
  - If the **entire** dropped batch is lossless: disable the slider (`.disabled(true)`, `.secondary` tint) and show the helper line (Copywriting below).
  - In a **mixed** batch: keep the slider enabled (applies to lossy members); each lossless row carries a "lossless" format tag and shows honest before/after size.
  - Never imply the slider shrinks PNG/TIFF by quality.

### Results Table — D-09 (net-new surface, no analog)

**One row per dropped image.** Constrained to popover width (~448pt usable). Column order, left → right:

1. **Thumbnail** — 40×40pt, `cornerRadius(4)`, fill-clipped. Rendered via `NSImage(contentsOf:)` (or ImageIO thumbnail API for large batches — 05-RESEARCH.md A1) in the View/ViewModel, never the Transformer. While loading: a `.quaternary` placeholder rectangle of the same size.
2. **Name + format tag** — filename (13pt, `.lineLimit(1)`, `.truncationMode(.middle)`) over a 10pt `.secondary` format tag ("JPEG", "PNG · lossless", "HEIC", "TIFF · lossless"). `VStack(alignment: .leading, spacing: 2)`.
3. **Size delta** — "{orig} → {new}" in 13pt monospaced `.secondary` (e.g. "2.4 MB → 1.1 MB"). Human-readable byte sizes via `ByteCountFormatter` (`.useMB/.useKB`, `.file` count style).
4. **% saved** — the hero metric, trailing-aligned. Caption 11pt semibold. `Color.green` "−{n}%" when saved; `Color.secondary` "+{n}%"/"0%" when grew/unchanged.

**Per-row states (D-09 live progress):**
- `.pending`: row visible with thumbnail + name; size/savings columns show a dim "—" placeholder.
- `.compressing`: indeterminate `ProgressView().scaleEffect(0.6)` (accent) in place of the size/savings columns.
- `.done`: size delta + `% saved` populated; rows update **live** as each finishes (the batch processes sequentially off-main, `await MainActor.run { rows[i].apply(result) }`).
- `.failed(reason)`: the row's size/savings columns are replaced by an inline `WarningBannerView(message: reason, severity: .warning)` (e.g. "Not a supported image" / "Couldn't write file"). `.error` severity only for a hard write failure.

**Single dropped file (discretion, D-09):** show the **same 1-row table** for consistency and less code (05-RESEARCH.md OQ-3) — do not build a separate single-file view.

**Cancellation:** while `isCompressing`, show a "Cancel" `Button` (`.foregroundStyle(.red)`, matches `HashView` L227–231) that calls `viewModel.cancel()`. Already-finished rows keep their results; pending rows stop.

**Output behavior (no UI surface, but informs copy):** each successful row writes `{name}-compressed.{ext}` beside the original (D-07), disambiguating to `-compressed-1`, `-2` if a collision exists (D-08). No save dialog. The destination is reflected only via the savings row, not a path display (keep the popover compact). Optionally each `.done` row may offer a "Reveal in Finder" affordance (`NSWorkspace.shared.activateFileViewerSelecting:`) — optional this phase.

### Empty State — D-05 consistency rule

Before any drop: the results area shows centered Body 13pt `.secondary` copy (see Copywriting) — never blank white space. The drop surface + quality controls remain visible above it.

### No Side-by-Side Comparison — D-10

No before/after visual comparison pane this phase. The table + `% saved` is the only feedback surface. Do not add a preview pane, a slider-reveal comparator, or a zoom view.

## Accessibility Contract

Carried from Phases 1–4 (VoiceOver labels on all interactive elements, Dynamic Type via `.system(size:weight:)` — no fixed-size text clips; CLAUDE.md Accessibility constraint). Phase 5 additions:

- Drop surface: `.accessibilityLabel("Drop images to compress")` (via `DropOverlayView`, existing).
- Quality slider: `.accessibilityLabel("Compression quality")` + `.accessibilityValue("{n} percent")`. When disabled (all-lossless batch): `.accessibilityHint("Disabled — the dropped images are lossless")`.
- Preset buttons: each `.accessibilityLabel("{name} quality preset")` + `.accessibilityHint("Sets quality to {value} percent")`. Active preset adds `.accessibilityAddTraits(.isSelected)`.
- Results table: each row is one combined accessibility element — `.accessibilityElement(children: .combine)` with `.accessibilityLabel` reading "{filename}, {format}, {state}". `.done` rows read "{filename}: saved {n} percent, {new size}". `.failed` rows read the failure message (via `WarningBannerView`'s existing combined a11y).
- Thumbnail `Image`/placeholder: `.accessibilityHidden(true)` (decorative; the row label conveys the file).
- Per-row `ProgressView`: `.accessibilityLabel("Compressing {filename}")`.
- "Cancel" button: `.accessibilityLabel("Cancel compression")`.
- Optional "Choose Images…" / "Reveal in Finder": explicit `.accessibilityLabel` on each.
