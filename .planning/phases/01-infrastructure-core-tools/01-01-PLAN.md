<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| clipboard → ClipboardDetector | Untrusted arbitrary user clipboard content crosses into detection + tool input |
| tool input → JSONTransformer | Untrusted/malformed/oversized text crosses into the pure transform layer |
| ViewModel → HistoryStore (SQLite) | Transformed data persisted to disk; must never include secrets |
| build config → code signing | Entitlements differ between debug and release; release must be notarization-safe |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-01-EP | Elevation of Privilege | Release entitlements | mitigate | Dual entitlements; `Lathe-release.entitlements` has NO `get-task-allow`; Hardened Runtime on; `CODE_SIGN_ENTITLEMENTS` per config. [BLOCKING] verified via grep in Task 1. |
| T-01-ID | Information Disclosure | HistoryEntry schema | mitigate | History schema has only id/tool/input/output/timestamp/pinned — no secret column exists by design (INFRA-09). JSON has no secret; the no-secret contract is frozen here for JWT/Hash. |
| T-01-DOS | Denial of Service | JSONTransformer on oversized/malformed input | mitigate | Pure `Result`-returning transformer, never force-unwraps; tested against 1MB random + malformed input (INFRA-17, Task 2). |
| T-01-SP | Spoofing | Clipboard auto-detect false positive | mitigate | Detection is non-destructive (banner only, never auto-opens — D-04); JSON pre-check requires `{`/`[` prefix before parse. |
| T-01-SC | Tampering | SPM package installs (GRDB, KeyboardShortcuts, MenuBarExtraAccess, HighlightSwift) | accept | All four [VERIFIED] approved in RESEARCH Package Legitimacy Audit; pinned to exact versions. nthState/UUIDV7 NOT installed in this plan (gated to 01-05). |
