<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| user search input → GRDB LIKE | Untrusted query string crosses into a SQL LIKE query |
| history restore → tool ViewModel | Saved input re-seeded into a tool; output recomputed (not trusted from disk) |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-06-T | Tampering (SQL injection) | History search LIKE query | mitigate | GRDB parameterized queries / query interface only; never string-interpolate the user query into SQL (GRDB API enforces binding). |
| T-06-IV | Input Validation | History restore re-running transforms | mitigate | Restore re-runs the pure Transformer on the saved input (D-08); stored output is preview-only and never trusted/executed. |
| T-06-ID | Information Disclosure | History contents | accept | History rows already exclude secrets by 01-01/01-03/01-04 design (INFRA-09); this plan only reads/deletes them. |
</threat_model>
