> Visual and interaction contract for Phase 1: Infrastructure + Core Tools.
> Generated by gsd-ui-researcher. Consumed by gsd-planner and gsd-executor.

> web stack, no shadcn, no Tailwind. All design tokens map to SwiftUI/AppKit system APIs.

## Spacing Scale

All values in SwiftUI logical points (pt). macOS HIG uses an 8-pt grid; values below follow
4-pt multiples per the GSD convention, mapped to common SwiftUI padding uses.

| Token | Value | SwiftUI Usage |
|-------|-------|---------------|
| xs | 4pt | Icon-to-label gap, inline badge padding, between pinned tool icons |
| sm | 8pt | `.padding(.horizontal, 8)` inside compact rows, detection banner vertical padding |
| md | 16pt | Default `.padding()` for tool content areas, list row horizontal padding |
| lg | 24pt | Section-level padding inside the detachable workspace window |
| xl | 32pt | Top/bottom padding for major layout regions (preferences pane sections) |
| 2xl | 48pt | Not used in Phase 1 popover (constrained to 480×600); reserved for workspace |
| 3xl | 64pt | Not used in Phase 1 |

**Exceptions:**
- Minimum tappable/clickable hit target: 44pt height for all interactive rows and buttons
  (macOS HIG accessibility requirement; aligns with INFRA-15 Dynamic Type).
- Pinned tool icon buttons in the quick-access row: 40×40pt touch target, 24pt SF Symbol
  rendering size, 4pt gap between icons (xs token).
- Popover fixed size: **480×600pt** (D-01, INFRA-01). No flex; this is the MenuBarExtra
  `.window` style constraint.
- Detachable workspace window minimum: **800×600pt** (INFRA-02).

## Typography

All sizes in points. SwiftUI Dynamic Type scales these relative to the user's system font
size preference (INFRA-15). Declare as `.font(.system(size: N, weight: .W))` or use
semantic styles where noted.

| Role | Size | Weight | Line Height | SwiftUI API | Usage |
|------|------|--------|-------------|-------------|-------|
| Body | 13pt | Regular (400) | 1.5 | `.font(.body)` or `.system(size: 13)` | Tool input/output text, history rows, search results |
| Label | 11pt | Regular (400) | 1.4 | `.font(.caption)` or `.system(size: 11)` | Metadata (byte count, char count, timezone labels, hash algorithm labels, expiry countdown) |
| Heading | 15pt | Semibold (600) | 1.2 | `.font(.headline)` or `.system(size: 15, weight: .semibold)` | Tool name in popover launcher, section headers in workspace sidebar |
| Code | 13pt | Regular (400) | 1.5 | `.font(.system(size: 13, design: .monospaced))` | JSON editor, Base64 output, JWT decoded payload, URL components, hash hex strings, UUID strings |

**Two weights only:** Regular (400) and Semibold (600). No bold (700), no medium (500).

**Monospace note:** Code role uses `.monospaced` design variant of San Francisco (SF Mono).
This is the system code font — no external font download required.

## Color

SwiftUI semantic colors adapt to Light/Dark mode automatically (INFRA-14). Never hardcode
hex values in SwiftUI layers — use semantic names. Hex equivalents are approximate reference
values for Light mode.

| Role | SwiftUI Value | Approx. Hex (Light) | Usage |
|------|--------------|---------------------|-------|
| Dominant (60%) | `Color(NSColor.windowBackgroundColor)` | #ECECEC | Popover background, workspace window chrome |
| Secondary (30%) | `Color(NSColor.controlBackgroundColor)` | #F5F5F5 | Tool content area background, list rows, preference panes, detection banner background |
| Accent (10%) | `Color.accentColor` (system accent) | varies | Reserved — see list below |
| Destructive | `Color.red` / `Color(NSColor.systemRed)` | #FF3B30 | "Clear History" button label, "Delete" history row action, destructive confirmation text |

**Accent color reserved for (explicit list — no other use):**
1. Primary CTA button background ("Copy Output", "Hash File", "Generate", "Now")
2. Accepted state of detection banner ("Open [Tool]" Accept button)
3. Search bar focus ring
4. Pinned history item pin icon (filled state)
5. Keyboard shortcut badge in the onboarding/preferences hotkey display

**Accent is NOT used for:** tool icons in the pinned row (use `.secondary` tint), error state
text, metadata labels, section headers, or toolbar icons.

**Error / warning colors:**
- Inline JSON error (JSON-03): `Color.orange` label text for the error annotation
  (orange, not red, because the output is not deleted — it is dimmed, and orange communicates
  "warning: input currently invalid" without implying data loss)
- JWT warning banners (JWT-06: expired, alg:none, missing claims): `Color.yellow`
  background tint on the banner row, `Color.primary` text
- JWT expired badge: `Color.red` text label ("EXPIRED")
- Hash file progress: `ProgressView()` with accent tint

## Interaction Contracts

These are the behavior rules the executor must implement. Each maps to one or more decisions
from CONTEXT.md.

### Popover Layout (D-01, INFRA-01)

Fixed 480×600pt MenuBarExtra `.window` style popover. Three zones stacked vertically:

1. **Search bar** — 44pt tall, full width, autofocused on every open. Placeholder: "Search tools or history…"
2. **Detection banner** — 0pt when no match, ~44pt when visible (slides down). Sits between search bar and pinned row.
3. **Pinned tool row** — 6 icons in a horizontal stack, 40×40pt each. Default order: JSON, Base64, JWT, URL, Timestamp, UUID (D-13).
4. **Body** — fills remaining space. Shows recent history (last 5 entries as compact rows) when search is empty and no tool is open. Shows search results list while typing. Shows tool view when a tool is selected.

### Navigation (D-02, D-03)

- Search bar persists at the top even when a tool view is displayed.
- Typing in the search bar while inside a tool view filters/switches tools (replaces the tool view with results list).
- **No back button.** Navigation is search-driven only.
- **Esc — two-stage:**
  - Stage 1: If inside a tool view, Esc returns to the launcher (empty search + pinned row + recent history). NavigationState resets to `.root`.
  - Stage 2: If already at launcher, Esc sets `isPresented = false` via MenuBarExtraAccess. Popover closes.
- Wire via `MenuBarExtraAccess` `isPresented` binding. `@Environment(\.dismiss)` must NOT be used — it has no effect inside `MenuBarExtra`.

### Detection Banner (D-04, D-05, D-06)

- Banner text format: "Detected: [Tool Name] — Open [Tool Name]?"
- Two buttons: **"Open [Tool Name]"** (accent fill button) and **"Dismiss"** (plain text button, .secondary color).
- Re-shown on every focus event when clipboard matches, even if dismissed last time. No per-value tracking.
- Single best match only (first-match-wins from predicate chain). No alternate match chips.
- Banner appearance: `RoundedRectangle(cornerRadius: 8)` with `Color(NSColor.controlBackgroundColor)` fill, 1pt border `Color.separator`. Slides in with `.transition(.move(edge: .top).combined(with: .opacity))` animation, `.animation(.easeOut(duration: 0.15))`.

### Live Transform + Debounce (D-10)

- **Live tools (150ms debounce):** JSON Formatter, Base64, URL Encoder/Decoder, JWT Decoder, Timestamp Converter, UUID Inspector.
- **Button-triggered tools:** Hash Generator file input (HASH-02), Base64 file encode/decode (B64-04), Bulk UUID generation (UUID-01 for counts > 1).
- Debounce implementation: `actor Debounce` with `Task.sleep(for: .milliseconds(150))` and task cancellation on new input. ViewModel concern, not View concern.

### Error State — Inline, Never Blank (D-11)

- On malformed input mid-typing: show inline error annotation with line:column (JSON-03) or generic message. Keep last valid output visible but **dimmed to 40% opacity** (`outputDimmed = true` → `.opacity(0.4)` on output view).
- Error annotation: small orange label `.font(.caption)` below the input field, text: "[Description] at line N, column M" (JSON), or "Invalid input" (other tools).
- Do NOT clear the output on error. Do NOT show a full-screen error state. This is mid-typing — the user is not done.
- When input becomes valid again, output opacity returns to 1.0 and error label disappears, all without animation (instant).

### Per-Field Copy Buttons (D-12)

Every output field that produces a discrete value has a copy button. Placement: trailing edge of the row, `Image(systemName: "doc.on.doc")` SF Symbol, `.foregroundColor(.secondary)`, 24×24pt hit area minimum.

Confirmed per-field copy locations:
- Hash Generator: one copy button per hash (MD5, SHA-1, SHA-256, SHA-384, SHA-512, CRC32) — 6 buttons (HASH-04)
- Timestamp: one copy button per timezone row (INFRA-02 / TS-02)
- URL Parser: one copy button per component (scheme, host, path, each query param, fragment) (URL-04)
- JWT: one copy button for header JSON, one for payload JSON, one for signature (JWT-01..02)
- UUID: one copy button per generated UUID row in bulk mode (UUID-04)
- JSON: "Copy Output" primary button for the full formatted result (JSON-06)
- Base64: "Copy Output" primary button for the encoded/decoded result

Copy feedback: button SF Symbol switches to `"checkmark"` for 1.5 seconds then reverts.
No toast/alert needed. Use `@State var copied: Bool` + `DispatchQueue.main.asyncAfter(deadline: .now() + 1.5)`.

### History Panel (D-07, D-08, D-09)

- Reachable via search query "history" or typing in search field.
- Shows last 100 unpinned + all pinned items. Pinned items sort to top (D-09).
- Compact row: Tool SF Symbol | Tool name | Input preview (truncated to 40 chars) | Relative timestamp | Pin icon | Delete button.
- Pin icon: `Image(systemName: "pin.fill")` in accent color when pinned; `Image(systemName: "pin")` in `.secondary` when unpinned.
- Clicking a row: loads the saved input into the matched tool, re-runs the transform live. Saved output is NOT restored — always recomputed (D-08).
- "Clear" button removes unpinned items only. Confirmation: inline popover "Clear 87 items? Pinned items will be kept." with "Clear" (destructive) and "Cancel".
- Individual row delete: swipe action or explicit delete button, no confirmation required (single item, recoverable by re-running).

## Accessibility Contract

Per INFRA-15 and macOS HIG:

- Every `Button`, `TextField`, `Toggle`, `NSTextView`, and interactive `List` row must have an `.accessibilityLabel()` or `setAccessibilityLabel()` (AppKit).
- `NSViewRepresentable` wrappers must call `textView.setAccessibilityLabel("Code editor")` and `textView.setAccessibilityRole(.textArea)` in `makeNSView`.
- VoiceOver announcement on copy: after 1.5s copy feedback, no secondary announcement needed — the button label change is sufficient.
- Detection banner must have `.accessibilityElement(children: .combine)` so VoiceOver reads the full suggestion as a unit.
- History rows must announce: "[Tool name], [input preview], [relative time], pinned/unpinned."
- All keyboard shortcuts listed in INFRA-16 must be set via `.keyboardShortcut()` or `KeyboardShortcuts` — never mouse-only.

## Visual Treatment: "Dimmed Last-Good Output" (D-11)

This is the central visual UX differentiator for Phase 1. Executor must implement exactly:

- Output area (`CodeDisplayView` or `Text` view for the result) wraps in `.opacity(viewModel.outputDimmed ? 0.4 : 1.0)`.
- No `.animation()` on the opacity change (instant is correct — flicker-free).
- The inline error label (`InlineErrorView`) appears/disappears with `.transition(.opacity)` and `.animation(.easeOut(duration: 0.1))`.
- The input field itself does NOT change color, background, or border on error — only the output dims and the error label appears below the input.
