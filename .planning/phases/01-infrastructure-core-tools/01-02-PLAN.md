<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| clipboard/user input → Base64/URL transformers | Untrusted arbitrary text + arbitrary files cross into the transform + file layers |
| file system → Base64 file encode/decode | Arbitrary (possibly large) files read/written without sandbox |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-SP | Spoofing | Base64 clipboard auto-detect false positive | mitigate | `isLikelyBase64` requires ≥12 chars + full base64 alphabet before suggesting decode (RESEARCH B64-03); detection is non-destructive banner only. Verified by test in Task 1. |
| T-02-DOS | Denial of Service | Base64 file encode (B64-04) on large files | mitigate | Chunked 1MB `FileHandle` read in `Task.detached`; never `Data(contentsOf:)` whole-file (pitfall #9 pattern). |
| T-02-IV | Tampering/Input Validation | URL/Base64 transformers on malformed input | mitigate | Pure `Result`-returning transformers; tested against malformed/garbage input (INFRA-17). |
| T-02-ID | Information Disclosure | HistoryEntry for both tools | accept | Base64/URL have no secret inputs; standard input/output history rows are expected and contain no credentials by nature of the tools. |
</threat_model>
