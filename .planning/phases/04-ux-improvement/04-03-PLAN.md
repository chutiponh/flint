<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| popover keyboard shortcut -> active tool (notification) | ⌘1–⌘9 broadcast a row index; only the mounted active tool view observes it. The index is untrusted in the sense that it may exceed a tool's row count. |
| tool output -> NSPasteboard | The selected row's text is written to the system clipboard. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-04-06 | Denial of Service | Out-of-range row index in .selectOutputRow handler | mitigate | `outputForRow(index)` returns nil for any index outside the tool's row map; the shared modifier no-ops indices ≠ 1; empty/nil output is a harmless no-op — never crash (CF-01, ASVS V5 input validation, RESEARCH Security Domain). |
| T-04-07 | Information disclosure | .selectOutputRow broadcast to a non-visible tool | accept | Only the mounted active tool view has a live `.onReceive` subscription; backgrounded tool views are not in the hierarchy (same mitigation as existing `.copyOutput`, ToolShortcutActions header T-09-03). |
| T-04-08 | Tampering | Writing wrong/unrelated data to clipboard | mitigate | Each handler writes only `viewModel.outputForRow(index)` (or `primaryOutput()`) of the active tool — never unrelated clipboard data; clears contents before setting (CopyButtonView pattern). No paste-back in this plan, so no synthetic events (CF-02 preserved). |
| T-04-SC | Tampering | npm/pip/cargo installs | accept | No package installs — no new SPM packages this phase (RESEARCH.md). Package Legitimacy Audit not applicable. |
</threat_model>
