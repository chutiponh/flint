## Spacing Scale

Carried forward from Phase 1 (01-UI-SPEC.md). All values in logical points, multiples of 4.

| Token | Value | SwiftUI Usage |
|-------|-------|---------------|
| xs    | 4pt   | Icon-to-label gap, badge inner padding, grid item spacing |
| sm    | 8pt   | Grid row spacing, banner vertical padding, back-header vertical inset |
| md    | 16pt  | Default `.padding()` for tool content areas; grid section horizontal padding |
| lg    | 24pt  | Onboarding window padding (established `OnboardingWindowView.padding(24)`); section-level padding |
| xl    | 32pt  | Not used in 480Ă—600 popover; workspace window major regions |
| 2xl   | 48pt  | Reserved; not used in Phase 4 new surfaces |
| 3xl   | 64pt  | Reserved; not used in Phase 4 new surfaces |

**Phase 4 exceptions / additions:**

- All-tools grid (`AllToolsGridView`): 3 adaptive columns, `GridItem(.adaptive(minimum: 100, maximum: 140), spacing: 8)`. Grid row spacing: 8pt (sm token). Grid outer padding: 12pt horizontal, 8pt vertical. 12pt is an off-scale inherited value matching the existing search bar and list row insets â€” do NOT normalize to 16pt this phase (would cause visual discontinuity against shipped components).
- Tool tile in the grid: `.padding(.vertical, 10)`. 10pt is an off-scale value acceptable for grid cells; closest scale value (8pt sm) feels cramped at 3 columns, closest above (12pt) is already the outer inset. 10pt is the confirmed design intent from RESEARCH.md Pattern 1.
- Back-affordance header (`ToolHeaderView`): 44pt minimum height (existing hit-target rule). Back button hit area: 44Ă—44pt. Horizontal padding: 12pt (matches existing list row inset).
- Number badge on output rows (D-08): 16Ă—16pt badge, 4pt inner padding, 4pt corner radius.
- Popover fixed size: **480Ă—600pt** (INFRA-01 â€” locked, unchanged).
- Workspace window minimum: **800Ă—600pt** (INFRA-02 â€” locked, unchanged).

## Typography

Carried forward from Phase 3 (03-UI-SPEC.md). Declared type scale: 4 sizes, 2 weights.

| Role | Size | Weight | Notes |
|------|------|--------|-------|
| Caption | 11pt | Semibold (600) | Number badge label, row index badge text â€” `.font(.system(size: 11, weight: .semibold, design: .monospaced))` |
| Body | 13pt | Regular (400) | Grid tool name, back-header title, onboarding refinement body text, update status messages â€” `.font(.system(size: 13))` |
| Subhead | 15pt | Semibold (600) | Tool tile name in grid (2-line max, centered), back-header tool name, onboarding section heads â€” `.font(.system(size: 15, weight: .semibold))` |

**Two weights only:** Regular (400) and Semibold (600). No bold (700), no medium (500).

**Phase 4 typography notes:**

- Grid tool name: use Body (13pt regular) inside each tile for legibility at the 100â€“140pt column width. Do NOT use Subhead â€” it overruns at 3 columns. `.lineLimit(2)`, `.multilineTextAlignment(.center)`.
- Back-header tool name: use Subhead (15pt semibold). This is a heading-level element that orients the user to which tool they are in.
- Update status text (in Preferences): use Body (13pt regular) for the status message (`.foregroundStyle(.secondary)` when idle/checking), Caption (11pt) is NOT used here â€” it is too small for an action-confirming state.
- Number badges (D-08): Caption (11pt semibold monospaced) inside the badge shape â€” compact and legible at the 16pt badge size.
- Off-scale inherited exceptions (same as Phase 3, locked): 10pt sub-labels, 12pt WarningBannerView text, 22pt SF Symbol glyph size.

## Interaction Contracts

These are the behavior rules the executor must implement exactly. Each maps to a decision from CONTEXT.md and supporting detail from RESEARCH.md.

### All-Tools Grid â€” D-01

**New surface:** `AllToolsGridView.swift` displayed at the `.root` navigation state below the pinned tool bar and above or replacing the recent history area when the user chooses to browse.

- Layout: `LazyVGrid` with `GridItem(.adaptive(minimum: 100, maximum: 140), spacing: 8)`. 12pt outer horizontal padding, 8pt outer vertical padding.
- Tiles: `VStack(spacing: 6)` â€” SF Symbol icon (22pt, accent) over tool name (13pt regular, 2-line max, centered).
- Tile background: `Color.quaternary.opacity(0.5)` fill, `cornerRadius(8)`.
- Tile tap action: `navigationState = .tool(toolId: tool.id)`.
- Tile hover state: `.background(.quaternary.opacity(0.85))` on hover (standard macOS hover tint; use `.onHover` + `@State` flag).
- 12 tiles total (all registered tools). Order: `toolRegistry.tools` order (existing registry order â€” do not reorder).
- Grouping: flat list, no category headers this phase (Claude's discretion confirmed in CONTEXT.md).
- Keyboard navigation: mouse/tab only this phase â€” arrow keys NOT wired to the grid (CONTEXT.md Deferred).
- VoiceOver: each tile has `.accessibilityLabel(tool.name)` and `.accessibilityHint("Open \(tool.name)")`.
- Performance: 12 items â€” `LazyVGrid` instantiation is instant; no lazy loading overhead.

**Placement in `MenuBarPopoverView`:** The all-tools grid is a persistent section visible in the `.root` state, sitting below the existing 6-pinned tool bar. It replaces the current empty/sparse space below the pinned row. Recent history (last 5 entries) remains accessible by scrolling below the grid within the root body.

### Back-to-Picker Affordance â€” D-02

**New component:** `ToolHeaderView.swift` wrapping every tool's `tool.makeView()` call in `bodyContent`.

- Layout: a `HStack` header pinned to the top of the tool area. Left-aligned back button + centered tool name.
- Back button: `Image(systemName: "chevron.left")` in `Color.accentColor` + `Text("All Tools")` in `Color.accentColor` at 13pt regular. Rendered as a `.plain` button style. `accessibilityLabel("Back to tool picker")`.
- Tool name: `Text(tool.name)` at 15pt semibold, `Color.primary`, centered in the remaining space.
- Header height: 44pt minimum.
- Separator: `Divider()` below the header, `Color(NSColor.separatorColor)`.
- Back tap action: `navigationState = .root`. This is the same target as Esc stage-1 (which is already wired). D-02 makes it explicit and persistent â€” both affordances coexist.
- The header wraps ALL 12 tools uniformly. It is NOT added inside individual tool views â€” it wraps `tool.makeView()` at the `bodyContent` switch site in `MenuBarPopoverView`.
- VoiceOver reading order: back button first, then tool name (left-to-right).

### Arrow-Key Search Navigation â€” D-07

**Existing code note:** `SearchView.swift` already has `@State selectedIndex`, `.onKeyPress(.upArrow/.downArrow/.return)`, and visual highlight on `isSelected`. This contract specifies the confirmed behavior, not new code (except the TextField focus verification).

- `â†‘` / `â†“` move `selectedIndex` through `flatResults`. Clamped: `selectedIndex` stays in `[0, flatResults.count - 1]`. Off-end presses are no-ops (no wrap-around).
- `Return` calls `activateSelected()` on the current `selectedIndex`.
- Visual highlight: row background `Color.accentColor.opacity(0.12)` when `isSelected`. Same opacity level as existing hover tints in the codebase.
- `selectedIndex` resets to `0` on every query change (`.onChange(of: query)` â€” already implemented).
- If `TextField` focus swallows arrow keys: escalate to `NSEvent.addLocalMonitorForEvents(matching: .keyDown)` for keyCodes 125 (â†“) and 126 (â†‘) in the search state, consuming the event and updating `selectedIndex`. This mirrors the existing `escMonitor` pattern. Do NOT add this preemptively â€” test empirically first.

### âŒ˜1â€“âŒ˜9 Row Copy â€” D-08

**New notification:** `Notification.Name("lathe.selectOutputRow")` with `userInfo["index"]: Int` (1â€“9). Added to `MenuBarPopoverView.swift` notification name extension block.

- Hidden buttons in `.background()` of `MenuBarPopoverView.body`: `KeyEquivalent(Character(String(index)))` for digits 1â€“9, `.command` modifier.
- Button action: posts `.selectOutputRow` notification with `index` value.
- Tool views observe: `.onReceive(NotificationCenter.default.publisher(for: .selectOutputRow))`. Handler: extract `index`, resolve the row value via `viewModel.outputForRow(index)`, write to `NSPasteboard.general`, clear prior contents first.
- Row index assignment (multi-output tools):
  - `ColorView`: row 1 = HEX, row 2 = RGB, row 3 = HSL, row 4 = HSV, row 5 = OKLCH
  - `HashView`: row 1 = MD5, row 2 = SHA-1, row 3 = SHA-256, row 4 = SHA-384, row 5 = SHA-512, row 6 = CRC32
  - `NumberBaseView`: row 1 = BIN, row 2 = OCT, row 3 = DEC, row 4 = HEX
  - All single-output tools: row 1 = their single copyable output (âŒ˜1 = copy output, mirrors existing âŒ˜â‡§C behavior)
- Number badge visual: 16Ă—16pt `RoundedRectangle(cornerRadius: 4)` with `.quaternary.opacity(0.6)` fill. Badge text: digit as 11pt semibold monospaced `.foregroundStyle(.secondary)`. Badge placed at the leading edge of the row, before the format label. Tooltip (`.help("Press âŒ˜\(index) to copy")`).
- Copy feedback: the row's existing `CopyButtonView` shows the `"checkmark"` for 1.5s (existing behavior). No additional animation on the badge itself.
- Out-of-range index (e.g. âŒ˜7 when tool has only 4 rows): silently no-op. Never crash (CF-01).
- Conflict check: âŒ˜1â€“âŒ˜9 are digit keys, NOT the letter N. `âŒ˜N` (letter) is the existing "Open workspace window" shortcut â€” unchanged. No conflict.
- Accessibility: badge has `.accessibilityLabel("âŒ˜\(index) to copy")`. Hidden buttons have `.accessibilityHidden(true)`.

### Update Checker â€” D-03 / D-04

**Surface:** `GeneralPreferencesTab` in `PreferencesView.swift`. A new "Updates" `Section` below existing general settings.

- Button: `Button("Check for Updatesâ€¦")`. `.disabled(sparkle.updateStatus == .checking)`. Style: default (not `.borderedProminent` â€” this is not a primary CTA; it is a utility action). `accessibilityLabel("Check for updates")`.
- Status display below the button (appears only when `updateStatus != .idle`):

  - `.upToDate`: `Label("Flint is up to date.", systemImage: "checkmark.circle.fill")` `.foregroundStyle(.green)` at 13pt

### Paste-Back Toggle â€” D-09

**Surface:** `GeneralPreferencesTab` in `PreferencesView.swift`. A new "Keyboard Flow" `Section`.

- Toggle: "Auto-paste result after copying". Default: **OFF** (`prefs.pasteBackEnabled` defaults to `false`).
- `.help("When enabled, pressing âŒ˜1â€“âŒ˜9 copies the result AND pastes it into the previously-focused app. Requires Accessibility permission.")`.
- `accessibilityLabel("Enable automatic paste-back after copying a result")`.
- Enable flow (toggle flips to ON): `if AXIsProcessTrusted() { prefs.pasteBackEnabled = true }` else call `AXIsProcessTrustedWithOptions` with `kAXTrustedCheckOptionPrompt: true`, then poll `AXIsProcessTrusted()` every 0.5s for up to 30s, then revert toggle if not granted.
- Revert copy: clear the toggle and show an inline message: `Text("Accessibility permission was denied. To enable paste-back, go to System Settings > Privacy & Security > Accessibility and add Flint.")` + `Button("Open System Settings") { NSWorkspace.shared.open(URL(string: "x-apple.systempreferences:com.apple.preference.security?Privacy_Accessibility")!) }`.
- When toggle is ON and permission is confirmed: `Text("Accessibility permission granted. âŒ˜1â€“âŒ˜9 will copy and paste the result into the previously-focused app.")` at 13pt `.secondary` (confirmation label below toggle).
- Paste-back action sequence (when `pasteBackEnabled && AXIsProcessTrusted()`): (1) write to `NSPasteboard.general`, (2) set `clipboard.isPopoverPresented = false`, (3) `app.activate(options: [.activateIgnoringOtherApps])`, (4) `DispatchQueue.main.asyncAfter(deadline: .now() + 0.08)` â†’ synthesize `CGEvent` âŒ˜V.
- Previously-focused app capture: in `HotkeyManager`, capture `NSWorkspace.shared.frontmostApplication` BEFORE posting `.showPopover` notification. Store as `var previousFrontmostApp: NSRunningApplication?` on `HotkeyManager`.
- At paste time: re-verify `AXIsProcessTrusted()` â€” if permission was revoked since toggle-on, fall back to copy-only silently (do not crash, do not paste).

### Visual Consistency Pass â€” D-05

This applies across all 12 tool views and the launcher. Normalized values:

- All tool content area padding: `md` (16pt). Any tool using 12pt or 10pt for content-area padding should be aligned to 16pt â€” unless the element is a row inset that matches existing shipped components (where 12pt is locked).
- Section dividers within tools: `Divider()` with `Color(NSColor.separatorColor)` â€” no custom colors.
- Empty/loading states: every tool must show a non-empty state when the input field is blank. Copy: "Paste or type content above" at 13pt `.secondary`, centered. No blank white space.
- Light/Dark: confirm all tool views use only semantic colors. Any `Color(hex:)` or hardcoded `.white` / `.black` literals must be replaced with `Color.primary` / `Color(NSColor.windowBackgroundColor)` respectively.
- Loading state for file operations (Hash file, Base64 file): always shows `ProgressView()` with a descriptive label â€” never shows a blank area.
